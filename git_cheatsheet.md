# 📜 Шпаргалка по Git (История версий)

Git позволяет сохранять "снимки" вашего кода (коммиты) и возвращаться к ним, если что-то пошло не так.

## 1. Как сохранить текущую версию (Сделать коммит)
Когда вы сделали что-то полезное и хотите "засохраниться":

1.  Откройте терминал.
2.  Введите команду, чтобы добавить все файлы:
    ```bash
    git add .
    ```
3.  Введите команду, чтобы сохранить с описанием (что сделали):
    ```bash
    git commit -m "Описание изменений"
    ```
    *Пример: `git commit -m "Добавил кнопку загрузки"`*

## 2. Как посмотреть историю (Лог)
Чтобы увидеть список всех сохранений:

```bash
git log --oneline
```
Вы увидите что-то вроде:
```
a1b2c3d Добавил кнопку загрузки
f4e5d6c Исправил ошибку в расчетах
9g8h7i6 Initial commit
```
*Слева — это "хеш" (ID) коммита.*

## 3. Как откатиться назад
Если вы все сломали и хотите вернуться к старой версии:

**Вариант А: Временно посмотреть старую версию**
```bash
git checkout <ID_коммита>
```
*Пример: `git checkout f4e5d6c`*
(Чтобы вернуться обратно к последней версии: `git checkout master`)

**Вариант Б: Жестко вернуть всё как было (Опасно! Удаляет новые изменения)**
```bash
git reset --hard <ID_коммита>
```
*Это удалит всё, что было сделано ПОСЛЕ этого коммита.*

## 4. Статус
Чтобы понять, есть ли несохраненные изменения:
```bash
git status
```
Если видите красные файлы — они изменены, но не добавлены. Зеленые — добавлены, но не закоммичены.
